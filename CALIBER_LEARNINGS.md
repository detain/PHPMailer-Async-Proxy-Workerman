Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[env]** `GITHUB_TOKEN` is set in this shell to an INVALID token — `gh` falls back to keyring login (account `detain`) only if `GITHUB_TOKEN` is unset for the call. Wrap every `gh` invocation as `unset GITHUB_TOKEN; gh ...` inline, otherwise `gh` will error with "The token in GITHUB_TOKEN is invalid".
- **[gotcha]** This fork lives NESTED inside the `mailbaby-mail-api` parent repo at `/home/sites/mailbaby-mail-api/PHPMailer-Async-Proxy-Workerman/`. Always `cd` into the fork (or use absolute paths) before running `git`/`composer` — running them from the parent operates on the wrong repo. Set `git config user.email` / `user.name` **locally inside the fork** (no `--global`), otherwise the parent repo's identity leaks in.
- **[convention]** Author identity for commits in this fork: `Joe Huss <detain@interserver.net>` (canonical — user types it inconsistently like "intertserver"; always normalize to `interserver.net`).
- **[pattern]** After editing `composer.json`, validate with `php -r 'json_decode(file_get_contents("composer.json"), false, 512, JSON_THROW_ON_ERROR);' && composer validate --no-check-all` before committing — catches JSON syntax + composer-schema errors in one shot.
- **[gotcha]** Pin `phpunit/phpunit` at `^9.6` ONLY, not `^9.6 || ^10.5`. `phpunit.xml.dist` uses the legacy PHPUnit 8.5 schema (`<listeners>`, `<filter>/<whitelist>`, `convertDeprecationsToExceptions`) and `test/DebugLogTestListener.php` `implements PHPUnit\Framework\TestListener` — PHPUnit 10 removed that interface, so the listener silently breaks. Same applies to `yoast/phpunit-polyfills`: stay on `^1.1`, not `^2.0` (which targets PHPUnit 10+).
- **[gotcha]** There is no committed `composer.lock`. Partial-update commands like `composer update phpunit/phpunit --with-all-dependencies` error with `Cannot update only a partial set of packages without a lock file present`. Run plain `composer update` (or `composer install` for fresh setups).
- **[gotcha]** Expect ~71 baseline test errors all reading `Test config params missing - copy testbootstrap-dist.php to testbootstrap.php`. They come from `SendTestCase::set_up()` and exist on master too — `test/testbootstrap.php` is gitignored. To eliminate, `cp test/testbootstrap-dist.php test/testbootstrap.php` and edit creds; otherwise treat that error count as the green baseline, not regression.
- **[pattern]** To prove failing tests are pre-existing (not caused by your branch): `git stash && git checkout master && composer install --no-interaction && vendor/bin/phpunit --no-coverage 2>&1 | tail -5` to capture baseline error count, then `git checkout <branch> && git stash pop`. Compare error totals — if equal/lower on your branch, the failures aren't yours.
- **[pattern]** Multi-step PR cadence in this fork: `git commit` → `git push -u origin <branch>` → `unset GITHUB_TOKEN; gh pr create --base master --head <branch> --title ... --body "$(cat <<'EOF' ... EOF)"` → `unset GITHUB_TOKEN; gh pr merge <PR#> --squash --delete-branch` → `git checkout master && git pull --rebase`. PR bodies use HEREDOC with backtick-escaped code spans (`\`name\``) so the shell does not interpret them.
