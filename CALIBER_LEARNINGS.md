# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[gotcha]** Workerman's Revolt-driver adapter file is `vendor/workerman/workerman/src/Events/Fiber.php` (`Workerman\Events\Fiber`), NOT `Events/Revolt.php` — the class name says `Fiber` because it relies on PHP Fibers, but it IS the Revolt bridge (`__construct()` does `$this->driver = EventLoop::getDriver()`). Use it as the global Workerman event-loop adapter when bootstrapping AsyncTcpConnection outside a real Worker (PHPUnit / CLI): `Worker::$globalEvent = new \Workerman\Events\Fiber()`. Re-bind it whenever the current Revolt driver changes (each `FiberRunner` private loop creates a fresh driver) — cache the driver you bound to and refresh when it differs. **`Workerman\Events\Fiber` ONLY ships in workerman/workerman 5.1+** — `^5.0` constraint can resolve to 5.0.x where the class doesn't exist and worker startup crashes. Use `"workerman/workerman": "^5.1"` in any composer.json that relies on this class.
- **[gotcha]** Composer 2.x rejects version strings like `8.0.0-async.0` / `-async.1` / `-async.2` with `Invalid version string` from `VersionParser.php` because the `-X` suffix triggers stability-keyword parsing and `async` is not a recognized keyword (only `alpha`, `beta`, `rc`, `dev`, `stable`, `patch`, plus `+meta` build metadata are accepted). Use **SemVer 2.0.0 build metadata** instead: `8.0.0+async.N`. Same precedence as `8.0.0`, but the `+async.N` suffix marks the fork iteration. Verify with: `cd /tmp && cat > composer.json <<<'{"name":"t/t","version":"X.Y.Z+meta"}' && composer --no-plugins validate --strict`. When bumping versions, update `VERSION` AND the three `const VERSION` declarations in `src/PHPMailer.php`, `src/SMTP.php`, `src/POP3.php` together — `test/ComposerMetadataTest::testVersionConstantsAreComposerNormalizable` will fail if they drift.
- **[convention]** A fork of a widely-used package MUST declare `"replace": {"upstream/pkg": "*"}` in `composer.json` so transitive deps that `require: upstream/pkg` resolve against the fork instead of pulling in upstream alongside (which would either double-install with namespace clash or fail outright). This fork has `"replace": {"phpmailer/phpmailer": "*"}` for exactly this reason. `test/ComposerMetadataTest::testReplaceDeclaresPhpMailerUpstream` locks the invariant down.
- **[pattern]** When fixing a contract that applies to multiple parallel implementations (e.g. `resolveImplicitCryptoMethod` across `StreamTransport`, `WorkermanConnectionTransport`, `WorkermanTransport`), audit ALL implementations BEFORE opening the PR — fixing two of three leaves the third as a latent bug that the next codex review round will flag. Search pattern: `grep -rn "function resolveImplicitCryptoMethod\|private function <name>" src/` to enumerate the surface, then update each in the same PR with a single shared unit test class that drives every implementation via reflection.
- **[gotcha]** When the deferred-TLS path manually drives `stream_socket_enable_crypto()` (because PROXY needs to be written before the handshake), it MUST honor `$contextOptions['ssl']['crypto_method']` (or the `tls` bucket) when the caller supplied one — PHPMailer exposes `SMTPOptions` for exactly this reason, and dropping it silently downgrades callers who locked the protocol set on stock PHPMailer. The fallback when nothing was supplied is `TLS_CLIENT | TLSv1_1_CLIENT | TLSv1_2_CLIENT`. See `StreamTransport::resolveImplicitCryptoMethod($contextOptions)`, `WorkermanConnectionTransport::resolveImplicitCryptoMethod($contextOptions)`, AND `WorkermanTransport::resolveImplicitCryptoMethod($contextOptions)` — all three transports share the contract, locked by `test/Async/CryptoMethodPassthroughTest.php` driving each private method via reflection.
- **[pattern]** When packaging-metadata bugs cause silent install failures (Composer rejects version string, namespace clashes from missing `replace`, runtime crashes from too-loose framework constraint), add a `test/ComposerMetadataTest.php` that decodes `composer.json` once via `set_up_before_class` and asserts each invariant: `replace` claims upstream, framework constraint regex matches the minimum-supporting line, VERSION file matches a Composer-acceptable regex AND mirrors the three `const VERSION` declarations. These bugs don't show up in unit tests of the code itself, but they break every downstream consumer — the test suite is where they belong.
- **[convention]** `src/POP3.php` follows the same Transport-injection pattern as `src/SMTP.php`: lazy `getTransport()` returning a `StreamTransport`, public `setTransport(Transport $t)`, and a private `buildErrorHandlerClosure()` for the protected-method-as-callable workaround. Don't re-introduce direct `fsockopen` / `fgets` / `fwrite` / `fclose` calls — POP-before-SMTP needs to be async-capable on the same terms as SMTP. Legacy `$pop_conn` is kept in sync with `getResource()` for fixtures that poke it directly, but new code should go through `$this->transport`.
- **[convention]** Fork-specific example scripts under `examples/` keep the `.phps` extension (web servers render source, don't execute) — `examples/workerman-async.phps`, `examples/proxy-protocol.phps`, `examples/connection-pool.phps`. Pattern: `require __DIR__ . '/../vendor/autoload.php';`, `use` statements for `PHPMailer\PHPMailer\Async\*` and `PHPMailer\PHPMailer\ProxyProtocol\*`, wire transport BEFORE `isSMTP()` / `setProxyProtocol()` so the lazily-created `StreamTransport` doesn't shadow your override, then run through `phpcs` (the `.phps` files are linted alongside `src/`).
- **[gotcha]** Expect ~74 baseline `markTestSkipped` results all reading `Live-SMTP test skipped: copy testbootstrap-dist.php to testbootstrap.php …`. They come from `SendTestCase::set_up()` and exist on master too — `test/testbootstrap.php` is gitignored. To eliminate, `cp test/testbootstrap-dist.php test/testbootstrap.php` and edit creds; otherwise treat 74 skipped as the green baseline. - **[gotcha]** A transport built on top of Workerman's `AsyncTcpConnection` CANNOT wrap each `Transport` method in its own `FiberRunner::run(...)` call. `AsyncTcpConnection` registers long-lived `onMessage` / `onClose` watchers in `connect()`; those watchers belong to whatever Revolt driver was active at registration time, and `FiberRunner::run()` swaps in a fresh private driver per invocation — subsequent `readLine()` calls fiber-suspend on a driver that never receives the readable events and the test hangs forever. The contract for any AsyncTcpConnection-backed transport is: caller wraps the WHOLE SMTP session in **one** `FiberRunner::run()`, the transport methods just `EventLoop::getSuspension()` directly.

Use `"workerman/workerman": "^5.1"`.
- **[pattern]** When a PHPUnit test touching `AsyncTcpConnection` / Revolt / fiber-suspension hangs and `timeout 30 vendor/bin/phpunit` produces an empty `{}` failure result, drop down to a standalone `php /tmp/diag-foo.php` script that replicates ONLY the transport + a `pcntl_fork`'d listener — no PHPUnit. The diag prints stepwise progress so you can see which call deadlocks and against which event-loop driver (`get_class(EventLoop::getDriver())`).
- **[gotcha]** In this Claude harness `BashOutput` / `KillShell` do not exist — the deferred tools for managing `run_in_background: true` bash tasks are `TaskOutput` (with `block: true, timeout: <ms>`) and `TaskStop`. Output for a background task is tail-able directly from `/tmp/claude-1000/<session>/tasks/<task_id>.output`. Use `ToolSearch` with a keyword query like `read background task output` to find the right tool names.
- **[gotcha]** PHPStan 1.x flags `private` properties that are assigned but never read (e.g. `errorSink` kept for API-symmetry with sibling transports) under rule `property.onlyWritten`. Silence with `@phpstan-ignore-next-line property.onlyWritten` immediately above the property — adding it to the class-level `ignoreErrors` in `phpstan.neon.dist` is overkill. Don't delete the property just to placate the linter when other transports in the family read it.
- **[gotcha]** `stream_socket_enable_crypto($sock, true, $method)` on a non-blocking stream returns THREE distinct values: `true` = handshake complete, `false` = handshake failed, `0` (integer) = would-block, kernel needs more I/O — caller MUST retry. The strict-equality check `=== 0` is required. The naïve "suspend on `onReadable` OR `onWritable`" pattern busy-loops because a connected TCP socket with send-buffer space is always writable. Use `onReadable` + a small **exponential-backoff timer (1ms → 50ms cap)** instead. See `WorkermanTransport::cryptoLoop()`.
- **[gotcha]** `WorkermanConnectionTransport::connect()` must branch on whether PROXY is enabled when the caller-supplied host carries an `ssl://` / `tls://` scheme: with PROXY → strip scheme, open `tcp://`, ship header, then upgrade manually via `cryptoLoop()`; **without PROXY → pass the `ssl://`/`tls://` URI through unchanged to `AsyncTcpConnection` so Workerman performs the native handshake**. Always coercing the URI to `tcp://` silently downgraded port-465 SMTPS to plaintext for non-PROXY callers.
- **[gotcha]** `convertDeprecationsToExceptions="true"` on the `<phpunit>` root attribute (a PHPUnit 9 default in older configs) turns every deprecation in dependencies into a hard test failure — including ones you have no control over. Remove the attribute (PHPUnit 10/11 default behavior is what you want) rather than chasing every upstream deprecation. Deprecation notices still surface in stderr where you can audit them, they just don't fail the suite.

- **[pattern]** Codex/ChatGPT bot PR reviews leave their **inline file:line comments** on the `comments` endpoint, NOT on the `reviews` endpoint that `gh pr view --json reviews,comments` returns. To extract: `gh api repos/OWNER/REPO/pulls/$PR/comments --jq '.[] | select(.user.login | test("chatgpt|codex"; "i")) | "FILE: \(.path):\(.line)\n\(.body)\n---"'`. The plain `gh pr view` output only shows the wrapper "💡 Codex Review" comment with no actionable content.
- **[convention]** `POP3::disconnect()` MUST call `$this->transport->close()` whenever a transport was ever built — even when `isOpen()` is already false because the peer hung up before QUIT. Skipping the close leaks the underlying stream resource until the next `connect()` or object destruction in long-running Workerman workers. The polite QUIT+response exchange goes inside an `if ($transport->isOpen()) {...}`, but the `close()` call itself sits outside it.
- **[gotcha]** `SmtpConnectionPool` examples that drive `PHPMailer::send()` MUST set `$mail->SMTPKeepAlive = true` BEFORE `send()` — otherwise `PHPMailer::smtpClose()` runs at the end of every send, issues QUIT, and closes the socket. The `$pool->release()` call in the `finally` block then receives a disconnected SMTP, `release()` drops it instead of caching, and every send pays the full TCP/TLS/AUTH handshake cost. Driving `$smtp->mail()`/`recipient()`/`data()` directly without going through `PHPMailer::send()` doesn't need the flag.
- **[gotcha]** PHPUnit `@group` annotations must live on the **class-level docblock**, NOT the file-level docblock at the top of the file. A `@group pop3` placed in the file header (before `declare(strict_types=1);`) is ignored — `--group pop3` filters the test out and `--list-tests --group pop3` shows it missing. Move the `@group` line into the docblock immediately preceding `final class FooTest extends TestCase`.
- **[gotcha]** `phpunit.xml.dist` excludes the `pop3` and `languages` groups by default (under `<groups><exclude>`). Running `vendor/bin/phpunit test/POP3/` reports `No tests executed!` and `--filter='POP3'` returns the same — those tests only run when you explicitly include the group: `vendor/bin/phpunit --no-coverage --group pop3`. The 6 POP3 tests come up to ~7s and confirm the StreamTransport path; CI workflows run them via the explicit `--group` flag.

- **[gotcha]** `git stash` (and `git stash push -u`) while in a conflicted merge state **drops `.git/MERGE_HEAD` and `.git/MERGE_MSG`**. After `git stash pop` you're no longer in a merge — `git commit` would create a regular commit with one parent, leaving the branch still reported as N-behind upstream. Recovery: `git stash push -u`, then `git merge -s ours --no-commit upstream/master` to recreate the merge state, then `git stash pop` to layer your resolutions, then `git add` + `git commit` produces a proper two-parent merge commit. Verify with `git log --format="%H %P" -1` — two parent hashes means the merge boundary is recorded.
- **[gotcha]** A `<<<<<<< HEAD` marker may already be present at HEAD from an earlier botched merge that someone committed with conflict markers intact. Before assuming all markers came from your current merge, `grep -n "<<<<<<<" file` against HEAD's version (`git show HEAD:file | grep`) — clean those up as part of the same merge resolution. Seen in this repo: `changelog.md` had two stacked `<<<<<<< HEAD` lines at the top, one from the current upstream merge plus one already on master.
- **[pattern]** To confirm "branch is no longer behind upstream" after a merge: `git log --oneline HEAD..upstream/master | wc -l` (behind) and `git log --oneline upstream/master..HEAD | wc -l` (ahead). GitHub's compare view doesn't update until you `git push origin master` — the local merge alone won't clear the "N commits behind" indicator. For a fork that GitHub recognizes the parent of, `gh repo view --json parent` returns the upstream owner/name.
