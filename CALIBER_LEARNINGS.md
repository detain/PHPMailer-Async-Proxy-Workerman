# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) ÔÇö do not edit manually.

- **[gotcha]** `stream_socket_enable_crypto($sock, true, $method)` on a non-blocking stream returns THREE distinct values, not just bool: `true` = handshake complete, `false` = handshake failed (cert/protocol mismatch), and **integer `0` = would-block, kernel needs more I/O ÔÇö caller MUST retry**. The strict-equality check `=== 0` is required; `!$ok` would treat would-block as failure. **Important nuance discovered after first wiring it:** the na├»ve "suspend on `onReadable` OR `onWritable`" pattern busy-loops because a connected TCP socket with send-buffer space is always writable, so `onWritable` fires immediately every iteration. Use `onReadable` + a small **exponential-backoff timer (1ms Ôćĺ 50ms cap)** instead ÔÇö `onReadable` covers the realistic blocker (server-hello / finished arriving) and the timer covers the rare "handshake wants more write room" case. See `WorkermanTransport::cryptoLoop()`.
- **[gotcha]** Self-signed `tls://127.0.0.1:0` listener + `pcntl_fork` + `stream_socket_accept` is FLAKY as a positive-path TLS test ÔÇö the child often exits with no accepted connection inside the timing window. Prefer testing the *would-block loop* itself: connect to a silent plain-TCP listener, call `enableCrypto()` with a 2s budget, and assert the call sat ~1.5ÔÇô4s (proving it suspended) and returned false. Real TLS handshake success is best verified against a live SMTPS test server in `MailTransportTest`, not a fork-and-accept in-process fixture.
- **[pattern]** PHPStan config in this fork lives at `phpstan.neon.dist` and scopes only the new async/proxy-protocol code: `paths: [src/Async, src/ProxyProtocol]`, `level: 5`, `treatPhpDocTypesAsCertain: false`, plus an `ignoreErrors` block silencing PHP stream-function `expects resource, ... given` false positives. Run with `vendor/bin/phpstan analyse --configuration phpstan.neon.dist --no-progress --memory-limit=512M`. Wired into CI via a separate `.github/workflows/static-analysis.yml` workflow, not the test matrix.
- **[gotcha]** `.gitignore` uses `docs/*` followed by explicit allowlist lines (`!docs/README.md`, `!docs/UPSTREAM.md`). Adding any new committed file under `docs/` requires another `!docs/<FILE>` line ÔÇö otherwise `git add docs/NEWFILE.md` silently does nothing and the next `git status` shows clean even though the file exists on disk. Same trap applies to `.claude` / `.caliber` etc. which are ignored at the dotfile level.
- **[gotcha]** PHPMailer (upstream) prepends `ssl://` (and sometimes `tls://`) onto `$host` when `SMTPSecure='ssl'`, so `stream_socket_client()` performs an implicit TLS handshake **before returning** ÔÇö which means any PROXY-protocol header would land AFTER the TLS ClientHello and port-465 receivers reject it. Both `StreamTransport::connect()` and `WorkermanTransport::connect()` work around this by: regex-stripping the scheme, connecting as plain `tcp://`, writing the PROXY bytes, then upgrading via `stream_socket_enable_crypto()` (or `cryptoLoop()` for the async path). Mirror that pattern in any new Transport implementation; see `test/Async/ProxyBeforeImplicitTlsTest.php` for the regression test.
- **[gotcha]** `SMTP::getSMTPConnection()` must return a TRUTHY value when the transport has no PHP stream resource (Workerman / coroutine / any wrapper around an OS-level socket that doesn't surface as a userland `resource`). Returning `false` because `$transport->getResource() === null` makes `SMTP::connect()`'s `=== false` check kill the session before the first protocol read. The fix is `return $resource ?? true;` ÔÇö and `SMTP::connected()` should call `$transport->isOpen()` instead of `is_resource($this->smtp_conn)`. See `test/Async/StreamlessTransportTest.php`.
- **[pattern]** When `FiberRunner` (or any helper that runs a private Revolt loop) is called from code that has *already* registered watchers / deferreds on the global driver but has NOT yet started the reactor, create a brand-new driver via `(new EventLoop\DriverFactory())->create()`, `EventLoop::setDriver($isolated)` for the duration of the run, and restore the previous driver in `finally`. Otherwise `$driver->run()` ticks the caller's pending callbacks (and our `$driver->stop()` kills them when our fiber finishes). See `FiberRunner::runWithPrivateLoop()` and the `testPrivateLoopDoesNotFireCallersPreScheduledCallbacks` regression test.
- **[pattern]** Setting GitHub repo metadata after a docs merge: `unset GITHUB_TOKEN; gh repo edit OWNER/REPO --description "..." --homepage "..." --add-topic foo --add-topic bar ...` ÔÇö repeat `--add-topic` per topic in a single call. Verify with `gh repo view OWNER/REPO --json description,homepageUrl,repositoryTopics`. Topics added via `--add-topic` are sorted alphabetically server-side, not preserved in insertion order.
- **[convention]** Auto-detect (`V1Header::auto()` / `Configurator::v1()`) now THROWS when the source IP is neither valid IPv4 nor IPv6 rather than silently degrading to `PROXY UNKNOWN\r\n`. The spec-mandated no-peer-info form is available via the explicit `V1Header::unknown()` factory (or by passing `FAMILY_UNKNOWN` to the constructor). Future callers passing user-supplied IPs to the auto-detect path should catch `ProxyProtocol\Exception` and decide between bubbling the error or falling back to `unknown()`.
- **[pattern]** When `phpcs` reports style errors marked `[x]` ("PHPCBF CAN FIX ÔÇŽ AUTOMATICALLY"), run `vendor/bin/phpcbf <path>` to auto-fix, then re-run `phpcs` to confirm clean ÔÇö don't hand-edit fixable issues. Warnings without `[x]` (e.g. "Line exceeds 120 characters") must be fixed manually; split long lines by extracting locals before passing to a call, rather than wrapping the existing expression.

- **[pattern]** Driving `STREAM_CLIENT_ASYNC_CONNECT` to completion: after `stream_socket_client(..., STREAM_CLIENT_ASYNC_CONNECT)` returns a resource, suspend on `EventLoop::onWritable($sock, ...)` until it fires, then verify success with `socket_import_stream($sock)` + `socket_get_option($s, SOL_SOCKET, SO_ERROR)` ÔÇö a non-zero SO_ERROR means the connect actually failed even though the socket is writable. `ext-sockets` is the gate here; when absent, the transport still works but can't surface the specific async-connect errno. - **[gotcha]** There is NO `composer.lock` and it is intentional ÔÇö `composer.json` has `"config": { "lock": false }` (plus `composer.lock` is in `.gitignore`). Partial-update commands like `composer update phpunit/phpunit --with-all-dependencies` error with `Cannot update only a partial set of packages without a lock file present`. **However, `composer require --dev <pkg>:<constraint>` works fine** in this no-lockfile setup ÔÇö that's the right command for adding new dev deps. Use plain `composer update` / `composer install` for everything else, and never check in a lockfile.
- **[gotcha]** `SMTP::connected()` must preserve upstream's three-state contract: EOF on a previously-open socket should emit the `SMTP NOTICE: EOF caught while checking if connected` debug line AND call `$this->close()` as a side effect, not just return false. The fork's transport-based rewrite originally dropped this because it short-circuited on `!$transport->isOpen()` before checking metadata. Correct shape: check `$transport->getMetadata()['eof']` FIRST, and gate the notice+close on whether there ever was a connection (`getResource() !== null || $this->smtp_conn`), then return `$transport->isOpen()` as the fallback.

- **[fix]** PHPUnit 9 prints `Warning: Your XML configuration validates against a deprecated schema. Suggestion: Migrate your XML configuration using "--migrate-configuration"!` on every run when the config root still references the 8.5 schema. Auto-fix is `vendor/bin/phpunit --migrate-configuration`, which writes the new format AND creates a `phpunit.xml.dist.bak` backup file you MUST `rm -f phpunit.xml.dist.bak` before committing (it's not in `.gitignore`). Re-run `vendor/bin/phpunit --no-coverage` and confirm the Warning + Suggestion lines no longer appear. Same flag works for `--migrate-configuration` on any PHPUnit 9.3+ config in this codebase.
- **[convention]** `src/POP3.php` now follows the same Transport-injection pattern as `src/SMTP.php`: lazy `getTransport()` returning a `StreamTransport`, public `setTransport(Transport $t)`, and a private `buildErrorHandlerClosure()` for the same protected-method-as-callable workaround. Don't re-introduce direct `fsockopen` / `fgets` / `fwrite` / `fclose` calls anywhere ÔÇö POP-before-SMTP needs to be async-capable on the same terms as SMTP. Legacy `$pop_conn` is kept in sync with `getResource()` for fixtures that poke it directly, but new code should go through `$this->transport`.

- **[gotcha]** A transport built on top of Workerman's `AsyncTcpConnection` (rather than raw `stream_socket_client` + Revolt) CANNOT wrap each `Transport` method (`connect` / `readLine` / `write` / `close`) in its own `FiberRunner::run(...)` call. `AsyncTcpConnection` registers long-lived `onMessage` / `onClose` watchers in `connect()`; those watchers belong to whatever Revolt driver was active at registration time, and `FiberRunner::run()` swaps in a fresh private driver per invocation ÔÇö so subsequent `readLine()` calls fiber-suspend on a driver that never receives the readable events and the test hangs forever (eats the PHPUnit 30s default and looks like a deadlock). The contract for any AsyncTcpConnection-backed transport is: caller wraps the WHOLE SMTP session in **one** `FiberRunner::run()`, the transport methods just `EventLoop::getSuspension()` directly. Inside a real running Workerman worker the loop is always alive so this is free.
- **[gotcha]** Workerman's Revolt-driver adapter file is `vendor/workerman/workerman/src/Events/Fiber.php` (`Workerman\Events\Fiber`), NOT `Events/Revolt.php` ÔÇö the class name says `Fiber` because it relies on PHP Fibers, but it IS the Revolt bridge (`__construct()` does `$this->driver = EventLoop::getDriver()`). Use it as the global Workerman event-loop adapter when bootstrapping AsyncTcpConnection outside a real Worker (PHPUnit / CLI): `Worker::$globalEvent = new \Workerman\Events\Fiber()`. Re-bind it whenever the current Revolt driver changes (each `FiberRunner` private loop creates a fresh driver) ÔÇö cache the driver you bound to and refresh when it differs.
- **[gotcha]** When a test fake subclasses `SMTP` and tracks call counts for `close()`, make the override **idempotent** (early-return when already closed). The connection pool's `safeClose()` calls `quit()` (which internally closes on success) and then `close()` again as belt-and-suspenders ÔÇö a naive `closeCount++` on every entry will double-count and break assertions. Mirror the real `SMTP::close()` shape: guard the body on `$this->connected` / equivalent state and flip the flag at the top.
- **[pattern]** When a PHPUnit test (especially anything touching `AsyncTcpConnection` / Revolt / fiber-suspension) hangs and `timeout 30 vendor/bin/phpunit` produces an empty `{}` failure result, drop down to a standalone `php /tmp/diag-foo.php` script that replicates ONLY the transport + a `pcntl_fork`'d listener ÔÇö no PHPUnit. The diag prints stepwise progress (`Connecting...`, `Connect returned: true`, `Trying readLine...`) so you can see which call deadlocks and against which event-loop driver (`get_class(EventLoop::getDriver())`). Running the full test runner masks where the suspension is parked.
- **[gotcha]** In this Claude harness `BashOutput` / `KillShell` do not exist ÔÇö the deferred tools for managing `run_in_background: true` bash tasks are `TaskOutput` (with `block: true, timeout: <ms>`) and `TaskStop`. Output for a background task is also tail-able directly from `/tmp/claude-1000/<session>/tasks/<task_id>.output`. `ToolSearch` with `select:BashOutput` returns no matches ÔÇö use a keyword query like `read background task output` to find the right tool names.
- **[gotcha]** PHPStan 1.x flags `private` properties that are assigned but never read (e.g. `errorSink` kept for API-symmetry with sibling transports) under rule `property.onlyWritten`. Silence with a `@phpstan-ignore-next-line property.onlyWritten` doc comment immediately above the property ÔÇö adding it to the class-level `ignoreErrors` in `phpstan.neon.dist` is overkill for a one-off. Don't delete the property just to placate the linter when other transports in the family read it.

- **[gotcha]** `WorkermanConnectionTransport::connect()` must branch on whether PROXY is enabled when the caller-supplied host carries an `ssl://` / `tls://` scheme: with PROXY Ôćĺ strip scheme, open `tcp://`, ship header, then upgrade manually via `cryptoLoop()`; **without PROXY Ôćĺ pass the `ssl://`/`tls://` URI through unchanged to `AsyncTcpConnection` so Workerman performs the native handshake**. - **[gotcha]** `convertDeprecationsToExceptions="true"` on the `<phpunit>` root attribute (a PHPUnit 9 default in older configs) turns every deprecation in dependencies into a hard test failure ÔÇö including ones you have no control over. Remove the attribute (the PHPUnit 10/11 default behavior is what you want) rather than chasing every upstream deprecation. After removal, deprecation notices still surface in stderr where you can audit them, they just don't fail the suite.
