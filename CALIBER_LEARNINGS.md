# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[convention]** Auto-detect (`V1Header::auto()` / `Configurator::v1()`) THROWS when the source IP is neither valid IPv4 nor IPv6 rather than silently degrading to `PROXY UNKNOWN\r\n`. The spec-mandated no-peer-info form is available via the explicit `V1Header::unknown()` factory (or by passing `FAMILY_UNKNOWN` to the constructor, or `Configurator::v1Unknown()` from PHPMailer-side wiring). Future callers passing user-supplied IPs to the auto-detect path should catch `ProxyProtocol\Exception` and decide between bubbling the error or falling back to `unknown()`.
- **[fix]** PHPUnit 9 prints `Warning: Your XML configuration validates against a deprecated schema. Suggestion: Migrate your XML configuration using "--migrate-configuration"!` on every run when the config root still references the 8.5 schema. Auto-fix is `vendor/bin/phpunit --migrate-configuration`, which writes the new format AND creates a `phpunit.xml.dist.bak` backup file you MUST `rm -f phpunit.xml.dist.bak` before committing (it's not in `.gitignore`). Re-run `vendor/bin/phpunit --no-coverage` and confirm the Warning + Suggestion lines no longer appear. Same flag works for `--migrate-configuration` on any PHPUnit 9.3+ config in this codebase.
- **[convention]** `src/POP3.php` now follows the same Transport-injection pattern as `src/SMTP.php`: lazy `getTransport()` returning a `StreamTransport`, public `setTransport(Transport $t)`, and a private `buildErrorHandlerClosure()` for the same protected-method-as-callable workaround. Don't re-introduce direct `fsockopen` / `fgets` / `fwrite` / `fclose` calls anywhere — POP-before-SMTP needs to be async-capable on the same terms as SMTP. Legacy `$pop_conn` is kept in sync with `getResource()` for fixtures that poke it directly, but new code should go through `$this->transport`.
- **[convention]** Fork-specific example scripts under `examples/` keep the `.phps` extension (web servers render source, don't execute) — `examples/workerman-async.phps` for the async transport wiring and `examples/proxy-protocol.phps` for the v1/v2/UNKNOWN/disabled config combinations. Pattern when adding similar demos: `require __DIR__ . '/../vendor/autoload.php';`, `use` statements for `PHPMailer\PHPMailer\Async\*` and `PHPMailer\PHPMailer\ProxyProtocol\*`, wire transport BEFORE `isSMTP()` / `setProxyProtocol()` so the lazily-created `StreamTransport` doesn't shadow your override, then run through `phpcs` (the `.phps` files are linted alongside `src/`).

They come from `SendTestCase::set_up()` and exist on master too — `test/testbootstrap.php` is gitignored. - **[gotcha]** A transport built on top of Workerman's `AsyncTcpConnection` (rather than raw `stream_socket_client` + Revolt) CANNOT wrap each `Transport` method (`connect` / `readLine` / `write` / `close`) in its own `FiberRunner::run(...)` call. `AsyncTcpConnection` registers long-lived `onMessage` / `onClose` watchers in `connect()`; those watchers belong to whatever Revolt driver was active at registration time, and `FiberRunner::run()` swaps in a fresh private driver per invocation — so subsequent `readLine()` calls fiber-suspend on a driver that never receives the readable events and the test hangs forever (eats the PHPUnit 30s default and looks like a deadlock). The contract for any AsyncTcpConnection-backed transport is: caller wraps the WHOLE SMTP session in **one** `FiberRunner::run()`, the transport methods just `EventLoop::getSuspension()` directly. Inside a real running Workerman worker the loop is always alive so this is free.
- **[gotcha]** Workerman's Revolt-driver adapter file is `vendor/workerman/workerman/src/Events/Fiber.php` (`Workerman\Events\Fiber`), NOT `Events/Revolt.php` — the class name says `Fiber` because it relies on PHP Fibers, but it IS the Revolt bridge (`__construct()` does `$this->driver = EventLoop::getDriver()`). Use it as the global Workerman event-loop adapter when bootstrapping AsyncTcpConnection outside a real Worker (PHPUnit / CLI): `Worker::$globalEvent = new \Workerman\Events\Fiber()`. Re-bind it whenever the current Revolt driver changes (each `FiberRunner` private loop creates a fresh driver) — cache the driver you bound to and refresh when it differs.
- **[gotcha]** When a test fake subclasses `SMTP` and tracks call counts for `close()`, make the override **idempotent** (early-return when already closed). The connection pool's `safeClose()` calls `quit()` (which internally closes on success) and then `close()` again as belt-and-suspenders — a naive `closeCount++` on every entry will double-count and break assertions. Mirror the real `SMTP::close()` shape: guard the body on `$this->connected` / equivalent state and flip the flag at the top.
- **[pattern]** When a PHPUnit test (especially anything touching `AsyncTcpConnection` / Revolt / fiber-suspension) hangs and `timeout 30 vendor/bin/phpunit` produces an empty `{}` failure result, drop down to a standalone `php /tmp/diag-foo.php` script that replicates ONLY the transport + a `pcntl_fork`'d listener — no PHPUnit. The diag prints stepwise progress (`Connecting...`, `Connect returned: true`, `Trying readLine...`) so you can see which call deadlocks and against which event-loop driver (`get_class(EventLoop::getDriver())`). Running the full test runner masks where the suspension is parked.
- **[gotcha]** In this Claude harness `BashOutput` / `KillShell` do not exist — the deferred tools for managing `run_in_background: true` bash tasks are `TaskOutput` (with `block: true, timeout: <ms>`) and `TaskStop`. Output for a background task is also tail-able directly from `/tmp/claude-1000/<session>/tasks/<task_id>.output`. `ToolSearch` with `select:BashOutput` returns no matches — use a keyword query like `read background task output` to find the right tool names.
- **[gotcha]** PHPStan 1.x flags `private` properties that are assigned but never read (e.g. `errorSink` kept for API-symmetry with sibling transports) under rule `property.onlyWritten`. Silence with a `@phpstan-ignore-next-line property.onlyWritten` doc comment immediately above the property — adding it to the class-level `ignoreErrors` in `phpstan.neon.dist` is overkill for a one-off. Don't delete the property just to placate the linter when other transports in the family read it.
- **[gotcha]** `SendTestCase::set_up()` now calls `markTestSkipped` (NOT `throw`) when `test/testbootstrap.php` is missing — so the dev-checkout baseline became **74 skipped, 0 errors** (was 71 errors). Any future learning or tooling that grep'd for the old `'Test config params missing'` error string needs updating: the message is now `'Live-SMTP test skipped: copy testbootstrap-dist.php to testbootstrap.php …'`. CI still exercises the full set by provisioning testbootstrap.php + an smtp-sink before running phpunit; only the dev experience changed.
- **[gotcha]** `stream_socket_enable_crypto($sock, true, $method)` on a non-blocking stream returns THREE distinct values, not just bool: `true` = handshake complete, `false` = handshake failed (cert/protocol mismatch), and **integer `0` = would-block, kernel needs more I/O — caller MUST retry**. The strict-equality check `=== 0` is required; `!$ok` would treat would-block as failure. **Important nuance:** the naïve "suspend on `onReadable` OR `onWritable`" pattern busy-loops because a connected TCP socket with send-buffer space is always writable, so `onWritable` fires immediately every iteration. Use `onReadable` + a small **exponential-backoff timer (1ms → 50ms cap)** instead. See `WorkermanTransport::cryptoLoop()`.
- **[gotcha]** `WorkermanConnectionTransport::connect()` must branch on whether PROXY is enabled when the caller-supplied host carries an `ssl://` / `tls://` scheme: with PROXY → strip scheme, open `tcp://`, ship header, then upgrade manually via `cryptoLoop()`; **without PROXY → pass the `ssl://`/`tls://` URI through unchanged to `AsyncTcpConnection` so Workerman performs the native handshake**. Always coercing the URI to `tcp://` (the original implementation) silently downgraded port-465 SMTPS to plaintext for non-PROXY callers. The three-way decision (scheme+PROXY → manual upgrade; scheme+no PROXY → native; no scheme → plain `tcp://`) is the contract — see the comment block at `src/Async/WorkermanConnectionTransport.php:connect()`.

The `examples/README.md` keeps fork-specific demos in a dedicated section above the upstream catalog.
- **[gotcha]** `convertDeprecationsToExceptions="true"` on the `<phpunit>` root attribute (a PHPUnit 9 default in older configs) turns every deprecation in dependencies into a hard test failure — including ones you have no control over. Remove the attribute (the PHPUnit 10/11 default behavior is what you want) rather than chasing every upstream deprecation. After removal, deprecation notices still surface in stderr where you can audit them, they just don't fail the suite.
- **[gotcha]** When the deferred-TLS path manually drives `stream_socket_enable_crypto()` (because PROXY needs to be written before the handshake), it MUST honor `$contextOptions['ssl']['crypto_method']` (or the `tls` bucket) when the caller supplied one — PHPMailer exposes `SMTPOptions` for exactly this reason, and dropping it silently downgrades callers who locked the protocol set on stock PHPMailer. The fallback when nothing was supplied is `TLS_CLIENT | TLSv1_1_CLIENT | TLSv1_2_CLIENT`. See `StreamTransport::resolveImplicitCryptoMethod($contextOptions)` and `WorkermanConnectionTransport::resolveImplicitCryptoMethod($contextOptions)`; the contract is locked by `test/Async/CryptoMethodPassthroughTest.php` driving the private method via reflection (`ReflectionMethod::setAccessible(true); ->invoke(null|$obj, [...])`) — there's no need to spin a real TLS server for this contract check.
- **[pattern]** Codex/ChatGPT bot PR reviews leave their **inline file:line comments** on the `comments` endpoint, NOT on the `reviews` endpoint that `gh pr view --json reviews,comments` returns. To extract the actual review feedback, use: `gh api repos/OWNER/REPO/pulls/$PR/comments --jq '.[] | select(.user.login | test("chatgpt|codex"; "i")) | "FILE: \(.path):\(.line)\n\(.body)\n---"'`. The plain `gh pr view` output only shows the wrapper "💡 Codex Review" comment with no actionable content — the suggestions live in `pulls/$PR/comments`.
- **[convention]** `POP3::disconnect()` MUST call `$this->transport->close()` whenever a transport was ever built — even when `isOpen()` is already false because the peer hung up before QUIT. Skipping the close (the early-return-on-EOF shape) leaks the underlying stream resource until the next `connect()` or object destruction in long-running Workerman workers; pre-refactor code did `fclose()` unconditionally and we have to preserve that semantic. The polite QUIT+response exchange goes inside an `if ($transport->isOpen()) {...}`, but the `close()` call itself sits outside it. See `test/POP3/Pop3DisconnectClosesTransportTest.php`.

- **[gotcha]** `SmtpConnectionPool` examples that drive `PHPMailer::send()` MUST set `$mail->SMTPKeepAlive = true` BEFORE `send()` — otherwise `PHPMailer::smtpClose()` runs at the end of every send, issues QUIT, and closes the socket. The `$pool->release()` call in the `finally` block then receives a disconnected SMTP, `release()` notices and drops it instead of caching, so `idleNow` stays at 0 and every send pays the full TCP/TLS/AUTH handshake cost. The alternate path — driving `$smtp->mail()`/`recipient()`/`data()` directly without going through `PHPMailer::send()` — doesn't need the flag because `smtpClose()` never runs. See `examples/connection-pool.phps`.

- **[gotcha]** PHPUnit `@group` annotations must live on the **class-level docblock**, NOT the file-level docblock at the top of the file. A `@group pop3` placed in the file header (before `declare(strict_types=1);`) is ignored — `--group pop3` filters the test out and `--list-tests --group pop3` shows it missing. The fix is to move the `@group` line into the docblock immediately preceding `final class FooTest extends TestCase`. Symptom in this session: my new POP3 test ran fine with `phpunit test/POP3/MyTest.php` but vanished under `--group pop3` until the annotation was relocated.

- **[gotcha]** Composer 2.x rejects version strings like `8.0.0-async.0` / `-async.1` / `-async.2` with `Invalid version string` from `VersionParser.php` because the `-X` suffix triggers stability-keyword parsing and `async` is not a recognized keyword (only `alpha`, `beta`, `rc`, `dev`, `stable`, `patch`, plus `+meta` build metadata are accepted). Use **SemVer 2.0.0 build metadata** instead: `8.0.0+async.N`. Same precedence as `8.0.0`, but the `+async.N` suffix marks the fork iteration. Verify with: `cd /tmp && cat > composer.json <<<'{"name":"t/t","version":"X.Y.Z+meta"}' && composer --no-plugins validate --strict`. When bumping versions, update `VERSION` AND the three `const VERSION` declarations in `src/PHPMailer.php`, `src/SMTP.php`, `src/POP3.php` together — `test/ComposerMetadataTest::testVersionConstantsAreComposerNormalizable` will fail if they drift.
- **[convention]** A fork of a widely-used package MUST declare `"replace": {"upstream/pkg": "*"}` in `composer.json` so transitive deps that `require: upstream/pkg` resolve against the fork instead of pulling in upstream alongside (which would either double-install with namespace clash or fail outright). This fork has `"replace": {"phpmailer/phpmailer": "*"}` for exactly this reason. `test/ComposerMetadataTest::testReplaceDeclaresPhpMailerUpstream` locks the invariant down.
- **[pattern]** When fixing a contract that applies to multiple parallel implementations (e.g. `resolveImplicitCryptoMethod` across `StreamTransport`, `WorkermanConnectionTransport`, `WorkermanTransport`), audit ALL implementations BEFORE opening the PR — fixing two of three leaves the third as a latent bug that the next codex review round will flag. Search pattern: `grep -rn "function resolveImplicitCryptoMethod\|private function <name>" src/` to enumerate the surface, then update each in the same PR with a single shared unit test class that drives every implementation via reflection.

- **[pattern]** When packaging-metadata bugs cause silent install failures (Composer rejects version string, namespace clashes from missing `replace`, runtime crashes from too-loose framework constraint), add a `test/ComposerMetadataTest.php` that decodes `composer.json` once via `set_up_before_class` and asserts each invariant: `replace` claims upstream, framework constraint regex matches the minimum-supporting line, VERSION file matches a Composer-acceptable regex AND mirrors the three `const VERSION` declarations. These bugs don't show up in unit tests of the code itself, but they break every downstream consumer — the test suite is where they belong.
